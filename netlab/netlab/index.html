    netlab

    netlabとは従来のGUIベースのラボを廃止し、「インフラストラクチャをコードとして」をコンセプトとしたネットワークラボツールです。netlabはPython3パッケージを使用して異なる環境を統合できる オーケストレーションツールでもあります。

    必須要件

    • Containerlab
    • Python3

    Containerlabを事前にインストールしておく必要があります。

    インストール手順: Containerlab Installtion

    Note

    Pythonはpython3 --versionで以下の表示になっていれば大丈夫です。 

    $python3 --version
Python 3.12.3

    Installation

    1. neltabをグローバルでインストール 

      pip install --break-system-packages networklab

    2. netlabに必要な依存関係のインストール 

      netlab install clab
netlab install ansible
netlab install ubuntu

    3. aptでrichをインストールするとFaliedするのでapt remove python3-richでアンインストール後、pipから再インストールします。 

      apt remove python3-rich
pip install --break-system-packages rich==13.7.1

    netlab versionコマンドでインストールされているかを確認 

    $ netlab version
netlab version 1.9.2-post1

    定義ファイルの作成

    netlabではデフォルトで使用するイメージを事前に定義する必要があります。

    自身のホームディレクトリ(home/xxx)に.netlab.ymlを作成します。 

    $ cat .netlab.yml

#ポートフォワーディングの設定
defaults.providers.clab.forwarded:
  ssh: 2222

tools.graphite.enabled: True
tools.edgeshark.enabled: True

devices.iosv:
 clab.image: vrnetlab/cisco_vios:15.9.3M8
devices.csr:
 clab.image: vrnetlab/cisco_csr1000v:17.03.08
    devices.xxxでデフォルトイメージが紐付けられるので、clab.imageでContainerlabでビルドしたイメージと明示します。

    サポートされているイメージは以下を参照
    URL: https://netlab.tools/platforms/

    Info

    GraphiteやEdgesharkはtools.edgeshark.enabled: Trueを有効にすることでノードデプロイ時に自動的にデプロイされます。

    ここまでの手順で環境構築は終了になります。

    ラボの作成

    netlabでは、トポロジはtopology.ymlに記述します。適当な場所にディレクトリを作成し、topology.ymlというファイルを作成しましょう。

    defaults.providers.clab.forwarded:
  ssh: 2222

provider: clab

nodes:
  R1:
    device: csr
  R2:
    device: iosv

links: [ R1-R2]
    nodes:内でデバイス名を定義し、device:でIOSvやCSR1000vを選択した状態でlinksにトポロジを記述します。

    netlab createでtopology.ymlからContainerlab用の定義ファイルを作成します。 

    $ ls
topology.yml

    $ netlab create
[CREATED] provider configuration file: clab.yml
[CREATED] transformed topology dump in YAML format in netlab.snapshot.yml
[CREATED] graphite configuration directory
[CREATED] graphite/graphite-default.json rendering "graphite" format
[GROUPS]  group_vars for all
[GROUPS]  group_vars for csr
[HOSTS]   host_vars for R1
[GROUPS]  group_vars for iosv
[HOSTS]   host_vars for R2
[CREATED] minimized Ansible inventory hosts.yml
[CREATED] Ansible configuration file: ansible.cfg

    ラボのデプロイ

    netlab up

    パケットキャプチャ

    パケットキャプチャはEdgesharkのGUIを介して各ノードのIFに対してパケットキャプチャを行うことができます。 WiresharkのGUIを呼ぶにはプラグインをインストールします。

    GUIへは以下のポートへアクセス

    URL: https://localhost:5001

    Note

    IPアドレスはポートフォワーディングされるのでlocalhostの部分は各自の環境に応じてIPアドレスを変えてください。
